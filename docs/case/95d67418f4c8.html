Independently Exposing Relays and Digital Inputs to Home Assistant

Case 95d67418f4c8 · tasmota

#home assistant#decouple#switch#relay#setoption73#setoption114#setoption19#switchmode#mqtt

Проблема

A user wanted to configure a Tasmota board with two relays and two digital inputs so that all four components would be independently available in Home Assistant. The problem was that triggering the physical inputs was also toggling the relays, which was not the desired behavior.

Рішення

After a suggestion to use `SetOption73 1`, the user found a working configuration. To decouple inputs (configured as Switches) from their corresponding relays and have them report their state to Home Assistant, several `SetOption` commands are needed. The user shared their final, successful configuration: SO19 0 // Disable Home Assistant auto-discovery for default components SO73 1 // Detach buttons/switches from relays SO114 1 // Allow sending switch state as separate MQTT message // GPIOs configured for relays and switches GPIO14 1216 // Relay1 GPIO5 256 // Relay2 GPIO0 163 // Switch3 switchmode3 1 // Set switch3 to follow mode GPIO12 162 // Switch4 switchmode4 1 // Set switch4 to follow mode This configuration, along with corresponding setup in Home Assistant, allows the switches to act as independent sensors.
