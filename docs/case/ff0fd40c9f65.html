Using MCP23017 I/O Expander with Tasmota on ESP32

Case ff0fd40c9f65 · tasmota

#MCP23017#I/O expander#custom firmware#compiling#ESP32#driver

Проблема

A user was trying to use an MCP23017 I/O expander with an ESP32 running the standard Tasmota firmware, but it was not being recognized automatically.

Рішення

Support for the MCP23017 I/O expander is not included in standard Tasmota binaries. A custom firmware must be compiled for the ESP32 with the necessary defines, such as `#define USE_MCP230xx` and `#define USE_MCP230xx_ADDR 0x20`. When troubleshooting, even if `i2cscan` detects the device, the `sensor29` command may fail if the wrong GPIO pins are used (standard pins are GPIO21 for SCL and GPIO22 for SDA on ESP32). To verify that the driver is correctly included in the firmware, use the `drivers` command in the console. If the driver is missing, the compilation was not successful. For improved I2C bus stability, it is also recommended to add 10k pull-up resistors to both the SCL and SDA lines, connecting them to the 3.3V rail.
