Automatización sin delays con múltiples triggers en Home Assistant

Case 7b38f53af040 · domotica_es

#Home Assistant#automatización#delays#triggers#paralelo#IDs#enchufe#domótica

Проблема

Un usuario busca una forma sencilla de encender un enchufe y apagarlo a los 30 segundos en una automatización de Home Assistant, pero sin usar "delays" para evitar que se solapen las llamadas de otros triggers que deben ejecutarse en paralelo.

Рекомендація (не підтверджено)

Para evitar que los 'delays' bloqueen la ejecución de otros triggers en una automatización, se puede configurar el modo de ejecución. Si la automatización está en modo 'single', cuando se dispara un trigger (ej. encender un enchufe con un delay de 5 minutos), se bloquea el resto de ejecuciones hasta que termine. Cualquier nuevo disparo de otro trigger durante este tiempo será ignorado. Para permitir que múltiples triggers funcionen concurrentemente sin esperar a que terminen los 'delays' de otros, se debe configurar la automatización en modo 'parallel' o separar cada lógica en automatizaciones distintas. Una alternativa para gestionar la concurrencia es usar IDs en los triggers.
