Copia de seguridad automática con disco desconectado en NAS

Case 0a88ab2e37b9 · naseros

#NAS#Backup#Copia de seguridad#Scripting#rsync#Almacenamiento

Проблема

Un usuario busca una solución de copia de seguridad automática para su NAS donde un disco secundario se conecte solo periódicamente para realizar el backup y luego se apague/desmonte para ser inaccesible.

Рекомендація (не підтверджено)

Para realizar copias de seguridad en un disco que permanezca inaccesible y apagado la mayor parte del tiempo, se proponen varias soluciones. A nivel de software, es posible usar un script (ej. en bash) para montar el disco, sincronizar los datos con `rsync`, y luego desmontarlo. Mientras está desmontado, el disco es inaccesible. Para un control físico del encendido/apagado: 1. **NAS Comercial**: La marca ASUSTOR ofrece la función "MyArchive", que permite tratar discos internos como extraíbles, montándolos y desmontándolos a voluntad. Sin embargo, esto no corta la alimentación eléctrica del disco. (Ver: https://www.youtube.com/watch?v=UwvJ-EyDKcU) 2. **Solución Universal/DIY**: No es posible controlar la alimentación de bahías internas de forma selectiva. La solución recomendada es utilizar un disco en una carcasa externa USB con su propia fuente de alimentación. La alimentación de esta carcasa se puede gestionar con un enchufe inteligente (domótica) para encenderla solo cuando sea necesario para la copia de seguridad. Se advierte no alimentar discos mecánicos directamente desde el puerto USB del NAS. Ejemplos de carcasas externas: https://es.aliexpress.com/item/1005005594204689.html, https://es.aliexpress.com/item/1005007786917996.html
