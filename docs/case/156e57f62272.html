Tasmota Devices Unavailable in Home Assistant due to MQTT Authorization Error

Case 156e57f62272 · tasmota

#Home Assistant#MQTT#Connection Error#Authorization#Password#Unavailable#Troubleshooting#SetOption19

Проблема

A user reported that their Tasmota plugs, connected to Home Assistant via MQTT, would become unavailable after a while. The devices were still accessible via their Web UI, SetOption19 was correctly set to 0, and the Tasmota integration was installed. The user tried restarting everything without success.

Рішення

After being prompted to check the MQTT broker logs in Home Assistant, the user found an error: "Unable to connect to the MQTT broker: Connection Refused: not authorised." This indicated an authentication problem. The user then realized they had made a mistake in the password field for the MQTT user in their Home Assistant configuration. Correcting the password resolved the issue and made the devices available again.
