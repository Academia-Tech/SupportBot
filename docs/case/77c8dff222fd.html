Puntos de acceso WiFi TP-Link Omada no funcionan tras configurar router ER605

Case 77c8dff222fd · naseros

#tp-link#omada#er605#eap650#oc200#wifi#pppoe#troubleshooting#vlan

Проблема

Un usuario ha configurado un router TP-Link ER605 en modo PPPoE para Movistar. La conexión a Internet por cable funciona correctamente, pero los puntos de acceso WiFi (modelos EAP250 y EAP650) no dan conexión. El usuario también dispone de un controlador Omada OC200 pero no sabe cómo empezar a configurarlo.

Рекомендація (не підтверджено)

Se diagnostica que el problema no está en el router TP-Link ER605 configurado en modo PPPoE con Movistar, sino en la configuración de los puntos de acceso (APs) o en la gestión centralizada. El internet funciona por cable, pero no por WiFi. Se ofrecen dos vías de solución: 1. **Configuración en modo Standalone**: Se sugiere revisar el manual de configuración de los APs (ej. EAP650: https://support.omadanetworks.com/es/document/4601/) para los primeros pasos y opciones. 2. **Configuración con Controlador Omada OC200 (Recomendado)**: Se aconseja usar el controlador OC200 para una gestión centralizada. Para ello, se debe reiniciar el router ER605 y los APs (EAP225 y EAP650) a valores de fábrica. La topología de red recomendada es: Internet > Router Omada ER605 > switch Omada (ej. ES208G) > todo conectado al switch: los APs y el OC200. Una vez conectados, se accede a la IP del controlador para adoptar los dispositivos. La configuración del controlador se aplicará automáticamente a los dispositivos adoptados. El controlador funciona localmente, incluso sin internet, pero se puede configurar el PPPoE de Movistar en el OC200 para asegurar la conexión. El usuario confirma que tiene un switch ES208G y un EAP225 compatible con el controlador. El usuario agradece la ayuda.
