Añadir una carpeta a un contenedor Docker en Synology sin poder editarlo

Case f9e4083501fa · naseros

#docker#synology#jellyfin#contenedor#volumen#carpeta#duplicar

Проблема

Un usuario necesita añadir una nueva carpeta (volumen) a un contenedor Docker de Jellyfin que se está ejecutando en un NAS Synology. La interfaz de gestión de Docker de Synology no le permite editar la configuración del contenedor en ejecución, solo le ofrece la opción de duplicarlo.

Рішення

La solución consiste en seguir la recomendación de la propia interfaz de Synology: duplicar el contenedor existente. Durante el proceso de duplicación, se puede modificar la configuración para añadir la nueva carpeta. Una vez creado el nuevo contenedor con la configuración correcta, se debe detener el contenedor antiguo, iniciar el nuevo y, tras verificar que todo funciona como se espera, se puede proceder a borrar el contenedor original. El usuario confirma que este método funcionó perfectamente.
