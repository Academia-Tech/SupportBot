Case 8d19ef20269e · naseros

#cgnat#pepephone#doble nat#modo bridge#unifi#troubleshooting#redes

Проблема

Un usuario configura su router de Pepephone (ZTE F6440) en modo bridge y conecta su propio gateway Unifi. Observa que su gateway recibe una IP del rango 10.x.x.x, a pesar de que su IP pública es otra, lo que le hace sospechar de doble NAT o CG-NAT.

Рішення

La causa más común de no poder abrir puertos es estar bajo CG-NAT, y la solución es contactar al proveedor para solicitar una IP pública. Específicamente para Digi, este servicio tiene un coste adicional de 1€/mes. Durante la resolución de un caso con un NAS Ugreen y Transmission, aunque el usuario confirmó tener IP pública, siguió experimentando problemas. Se le proporcionó una guía detallada de configuración de red (cambiar la IP del router a 192.168.0.1, configurar DHCP, etc.), pero el usuario continuó teniendo dificultades para seguir los pasos y finalmente decidió posponer la solución para otro día, agradeciendo la ayuda recibida.
