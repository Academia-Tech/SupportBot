Relay on NodeMCU Toggles On But Fails to Turn Off

Case f6596f7aa96a · tasmota

#relay#nodemcu#gpio#troubleshooting#hardware#voltage level#3.3v#12v

Проблема

A user connected a relay to the D4 port of a NodeMCU running Tasmota. The relay successfully turns on at startup but does not respond to toggle commands to turn it off.

Рекомендація (не підтверджено)

Users diagnosed an issue where a 5V relay module (identified as a JQC-3FC(T73)) connected to a NodeMCU would turn on but not off. The root cause was determined to be a logic level/drive current issue, as the relay required more than the 3.3V provided by the ESP's GPIO to trigger correctly. The original poster confirmed they solved the issue by switching from Tasmota to ESPHome and using a specific "tri state custom relay component" from a GitHub repository (`https://github.com/ssieb/custom_components`). This indicates the relay has special control logic that was not easily handled by standard Tasmota configurations. Other suggestions included ensuring the relay was powered directly from a 5V source and using a transistor to provide sufficient drive current.
