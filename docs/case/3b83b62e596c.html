Неправильний порядок та напрямок обертання моторів і відсутність доступу до конфігуратора AM32

Case 3b83b62e596c · ua_ardupilot

#motor order#servo output#AM32#ESC configurator#SERVO_BLH_AUTO#passthrough#ArduPilot#Betaflight

Проблема

Користувач зіткнувся з двома проблемами: 1) Неправильний порядок моторів (4, 1, 2, 3 замість стандартного) та їх обертання в неправильну сторону. 2) Неможливість підключитися до конфігуратора AM32 для зміни налаштувань ESC через Mission Planner.

Рішення

Проблему було вирішено наступним чином: 1. **Порядок моторів:** В налаштуваннях Mission Planner для параметрів Servo Output (SERVOx_FUNCTION) було встановлено стандартний порядок 1, 2, 3, 4. 2. **Доступ до конфігуратора AM32:** Встановлення параметра `SERVO_BLH_AUTO` в значення `1` дозволило отримати доступ до налаштувань ESC через passthrough. 3. **Додаткові поради:** Для доступу до ESC може знадобитися повне перезавантаження польотного контролера, оскільки порт може бути заблокований після налаштування. Альтернативний метод - використовувати окремий польотний контролер з прошивкою Betaflight для конфігурації ESC.

Але вони крутяться не в ту сторону, бо в ЄСЦ мабуть виставлений реверс (чомусь не можу зайти в AM32 для зміни напрямку обертів).
Пішов іншим шляхом і виставив FRAME_TYPE 18, але тепер порядок моторів не відповідає картинці.
