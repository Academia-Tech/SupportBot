Using RCWL-0516 Motion Sensor with Tasmota Without Controlling the Relay

Case bd33d77f909f · tasmota

#rcwl-0516#motion sensor#home assistant#tasmota rules#decouple switch#scripting#sonoff basic#virtual relay

Проблема

A user connected an RCWL-0516 presence sensor to a Sonoff Basic. They want to see the sensor's state (high/low) in Home Assistant for automations, but when they configure it as a switch or button, it constantly toggles the device's physical relay. The goal is to decouple the sensor from the relay and simply read its state.

Рекомендація (не підтверджено)

A user provided several suggestions and workarounds to solve this: 1. **Detach the switch:** Use Tasmota commands to decouple the switch input from the relay. The documentation for this is at `https://tasmota.github.io/docs/Buttons-and-Switches/#detach-switches-with`. 2. **Home Assistant Configuration:** The problem might need to be solved within Home Assistant by configuring the switch entity as a binary sensor. 3. **Virtual Relay Workaround:** As a workaround, one can configure a "virtual" relay in Tasmota. This allows the sensor state to be reported to Home Assistant via MQTT without controlling a physical relay. 4. **Tasmota Scripting:** A final suggestion was to use Tasmota's scripting support. This allows for custom logic to define and display any value, including a switch's state, in the Tasmota Web UI, which can then be sent to Home Assistant.
