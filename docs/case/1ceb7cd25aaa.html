Troubleshooting Issues with Multiple DS18B20 Sensors on ESP8266

Case 1ceb7cd25aaa · tasmota

#DS18B20#ESP8266#D1 Mini#GPIO#multiple sensors#ESP32#XIAO ESP32C3#star topology

Проблема

A user is experiencing issues when connecting more than four DS18B20 waterproof temperature sensors to a single GPIO pin (GPIO0) on a D1 mini (ESP8266). The setup uses a star topology, a 4.7kΩ pull-up resistor, and an external 5V power supply. The user is unsure if the problem is electrical (power drop) or a software race condition.

Рекомендація (не підтверджено)

It is highly recommended to avoid using GPIO0 for sensors due to its special functions during boot. A user confirmed successfully running up to 10 DS18B20 sensors on pin D7 (GPIO13). For setups with many sensors, it is suggested to consider using an ESP32-based board, like the compact XIAO ESP32C3, which offers more flexible GPIO options. The original poster later confirmed that their problem was indeed related to using GPIO0 and that they were able to fix it.

Checkout this here...

GPIO15 is always pulled low, so you can’t use the internal pull-up resistor. You have to keep this in mind when using GPIO15 as an input to read a switch or connect it to a device with an open-collector (or open-drain) output, like I²C.
GPIO0 is pulled high during normal operation, so you can’t use it as a Hi-Z input.
GPIO2 can’t be low at boot, so you can’t connect a switch to it.
