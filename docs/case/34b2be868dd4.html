Troubleshooting DS18B20 sensors on long cable runs

Case 34b2be868dd4 · tasmota

#DS18B20#long cable#3.3V#5V#Logic Level Converter#LLC#sensor

Проблема

A user is experiencing issues with DS18B20 temperature sensors connected via long cables (3 and 5 meters) running on 3.3V.

Рекомендація (не підтверджено)

It was noted that according to documentation, DS18B20 sensors may be unreliable on cable lengths over 1 meter when powered by 3.3V. The recommended solution for longer runs is to power the sensors with 5V and use a Logic Level Converter (LLC) to safely interface the 5V sensor data line with the 3.3V GPIO pin of the microcontroller. Further user experiences confirm that long runs are possible with specific setups: * One user reported a reliable connection on a 10m+ non-shielded cable, suggesting it can work if electrical noise is low. * Another user shared a successful implementation for a 20-meter line with 6 sensors: power the sensors (VCC) with 5V, and use an 820 ohm pull-up resistor on the data line to 3.3V. This setup used 0.5mm section cables connected to a Raspberry Pi, highlighting that a stronger pull-up (a lower resistance value than the typical 4.7kΩ) can be key for stability on long lines.
