Cómo asignar una IP estática a un Shelly y debate sobre mejores prácticas de red

Проблема

Un usuario necesita asignar una IP estática a un dispositivo Shelly para controlarlo por HTTP, ya que con IP dinámica tiene problemas. Esto deriva en un debate sobre cuál es el mejor método para gestionar las IPs en una red domótica.

Рішення

Se presentan dos métodos principales para asignar una IP estática, junto con un debate sobre sus ventajas y desventajas: 1. **Desde la interfaz web del Shelly:** Es el método más directo. Se accede a la IP del dispositivo, se va a la configuración de red y se asigna una IP fija, máscara de red y puerta de enlace. Es crucial elegir una IP que esté fuera del rango que el router asigna por DHCP para evitar conflictos. 2. **Desde el router (Reserva DHCP):** Considerado la mejor práctica. Se accede a la configuración del router y se crea una reserva de IP para la dirección MAC del Shelly. Esto centraliza la gestión de la red, previene conflictos y facilita la administración de múltiples dispositivos. **Consideraciones adicionales:** Se advierte sobre el uso de redes de "invitados", ya que a menudo tienen el "aislamiento de clientes" activado, impidiendo que los dispositivos se comuniquen entre sí. También se recuerda que la IP .255 es de broadcast y no debe usarse.
