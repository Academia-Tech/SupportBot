Проблема

A user asked about the possibility of flashing Tasmota onto a WVC series solar inverter (e.g., WVC-600) to remove its dependency on a Chinese cloud service, noting the Wi-Fi module appeared to be ESP32-based.

Рекомендація (не підтверджено)

Another user strongly advised against purchasing or using WVC inverters, calling them a "scam". They explained that the devices are significantly underpowered (e.g., a 1600W model is needed for 800W of actual operation), have very poor MPPT (Maximum Power Point Tracking) implementation leading to overheating and shutdowns, and are generally unreliable. A follow-up investigation by user 'user_70d4e26e86' on the Wi-Fi enabled models revealed they use an LPT270-0 module based on a "GENIE+" chip, which cannot be reflashed with Tasmota. However, the communication protocol uses a simple set of AT commands. All inverter reports are sent via two main commands: `AT+SENDICA=property,PV_Volt,19.0,PV_Current,0.00,...` `AT+SENDICA=property,PowerSwitch,0,Plant,0.00,...` This information could be used to intercept and monitor the inverter's data with a separate ESP device.

Short update to WIFI enabled WVC inverters: They use an LPT270-0 WIFI/Bluetooth module that is based on a GENIE+ chip (never heard of that manufacturer) - so sadly this cannot simply be reflashed...

But they use a relatively simple set of AT commands to the module, while the module prefixes every message with a '+' character, replies every command back and sends "+ok" or "+ok=send failed" and sends its WIFI and server connection status updates as "+ILOPCONNECT=WIFI_CONNECT" / "+ILOPCONNECT=WIFI_CONNECT"

All the inverter reports are the following two messages:
AT+SENDICA=property,PV_Volt,19.0,PV_Current,0.00,PV_Power,0.0,AC_Volt,0.0,AC_Current,0.00,Out_Power,0.0,Temperature,34.0,Power_adjustment,100,Energy,0.00
AT+SENDICA=property,PowerSwitch,0,Plant,0.00,Emission,0.00,Time,30,P_adj,0,TEMP_SET,66
