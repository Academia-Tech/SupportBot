How to flash a custom compiled firmware .bin file to an ESP32

Case a43143e21c13 · tasmota

#esp32#flash#custom firmware#esptool.py#ota#tasmotizer#compile#rules#flash address

Проблема

A user needed to flash a custom compiled Tasmota `.bin` file to an ESP32 to enable features like `if` conditions. They were unsure how to do this, as Tasmotizer doesn't support ESP32 and OTA updates with the custom file failed. They specifically needed to know the correct command and flash address for `esptool.py`.

Рішення

To flash a custom firmware binary to an ESP32 that already has a bootloader, use the command: `esptool.py write_flash 0x10000 ./custom_XXX.bin`. The bootloader itself would be flashed to address `0x0` if partition changes are needed. **Update:** A user reported that even after successfully writing the firmware, the device entered a boot loop. The discussion continued with another user stating that the provided entries were incorrect and shared an example image from a different project (`https://github.com/martin-ger/esp32_nat_router/blob/master/FlasherUI.jpg`), indicating the solution may not be universally applicable and might be incorrect.
