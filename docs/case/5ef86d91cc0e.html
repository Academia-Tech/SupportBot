Неправильна орієнтація акселерометра на Speedybee F405 v4 в ArduPilot

Case 5ef86d91cc0e · ua_ardupilot

#AHRS_ORIENTATION#Speedybee F405 v4#accelerometer#orientation#IMU#hwdef#ArduPilot

Проблема

Користувач зіткнувся з проблемою, коли при нахилі дрона вліво, Mission Planner показує нахил вправо. Польотний контролер Speedybee F405 v4 встановлено стандартно, стрілкою вперед. Спроби змінити параметр AHRS_ORIENTATION (наприклад, Yaw 180) призводили до інших проблем з орієнтацією.

Рішення

Для польотного контролера Speedybee F405 v4, навіть при стандартному встановленні, потрібне специфічне значення параметра орієнтації. Згідно з файлом апаратної конфігурації (hwdef) ArduPilot, для параметра `AHRS_ORIENTATION` необхідно встановити значення `PITCH_180_YAW_270`. Також було зазначено, що візуалізація в Mission Planner показує "погляд" дрона, що може відрізнятися від звичного відображення в Betaflight.
