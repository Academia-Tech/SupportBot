Understanding SetOption132 and TLS for Sending Telegram Messages

Case abcc43cdf9d5 · tasmota

#tasmota#tls#ssl#telegram#setoption132#security#compilation#esp8266

Проблема

A user claimed that `SetOption132 1` is required to send Telegram messages in recent Tasmota versions. This led to a discussion about the true purpose of this setting and how to properly configure TLS for secure connections.

Рішення

A user clarified that `SetOption132` controls the TLS server validation method, not Telegram functionality directly. - `SetOption132 1` enables fingerprint validation, where Tasmota saves the server's TLS certificate fingerprint on first contact. This is a less secure but easier method that works out-of-the-box. - `SetOption132 0` (the default) attempts full certificate validation. On ESP8266, this requires compiling a custom firmware with the necessary SSL libraries included, as explained in the 'Compiling TLS for ESP8266' section of the Tasmota TLS documentation (https://tasmota.github.io/docs/TLS/). The issue with Telegram failing is likely due to an incomplete TLS setup when using `SetOption132 0` on a standard build.
