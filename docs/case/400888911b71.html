How to Configure Independent Relays and Inputs for Home Assistant

Case 400888911b71 · tasmota

#Home Assistant#Tasmota#relay#switch#input#decouple#SetOption73#SetOption114#MQTT

Проблема

A user wanted to configure a Tasmota device with two relays and two digital inputs so that all four components would be independently available in Home Assistant. The inputs were incorrectly toggling the relays instead of acting as separate entities.

Рішення

After initial suggestions, the user found a working configuration. The key was to use several `SetOption` commands to decouple the inputs from the relays and configure the device for Home Assistant discovery. The final Tasmota configuration that solved the issue was: SO19 0 SO26 1 SO73 1 SO114 1 GPIO14 1216 GPIO5 256 GPIO4 225 GPIO0 163 switchmode4 1 GPIO12 162 switchmode3 1 This was combined with custom friendly names, web buttons, and some configuration within Home Assistant.

To ignore default button behaviour of controlling power outputs you can:

use SetOption73 1
use ButtonTopic
define a rule which triggers on Button<x>#State. Take note: If the rule trigger only certain states, default behaviour is suppressed only for those states.
