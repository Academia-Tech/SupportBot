Unstable Input Signal from Relay Auxiliary Contact on Wemos

Case 665ddb916984 · tasmota

#debounce#ghost switching#noise#capacitor#input stability#Wemos#relay feedback

Проблема

A user is trying to get feedback from a commercial relay's auxiliary contact connected to a Wemos D1 Mini running Tasmota. The input signal is unstable, rapidly toggling on and off, which is likely due to electrical noise (e.g., 50/60Hz hum) being picked up.

Рекомендація (не підтверджено)

Several solutions were proposed to fix the unstable input from a relay's auxiliary contact on a Wemos board: 1. **Hardware Filtering:** Add a decoupling capacitor (e.g., 10µF) between the GPIO input pin and Ground to filter out high-frequency noise. 2. **Hardware Pull-up:** Add a 10k pull-up resistor from the input pin (D1) to the 3.3V line to ensure a stable HIGH state when the contact is open. A link explaining pull-up resistors was provided: https://www.electronics-tutorials.ws/logic/pull-up-resistor.html 3. **Software Debounce:** Increase the software debounce timer in Tasmota. The documentation for "ghost switching" on Shelly devices was provided as a reference: https://tasmota.github.io/docs/devices/Shelly-2.5/#ghost-switching 4. **Software Pull-up:** Use the internal pull-up resistor by setting `pinMode(D1,INPUT_PULLUP);` in the code. However, it was noted this might not be directly configurable in Tasmota for general inputs, as `Setoption74` is specific to DS18B20.
