Error 'Adapter EZSP protocol version not supported' en Zigbee2MQTT con Sonoff Dongle-E

Case cf4f4694dbcc · domotica_es

#Zigbee2MQTT#Z2M#Sonoff#ZBDongle-E#firmware#flashear#EZSP#Ember#Home Assistant

Проблема

Un usuario, al configurar Zigbee2MQTT por primera vez en un miniPC con Home Assistant, se encuentra con un error en los logs: "Adapter EZSP protocol version (8) is not supported by Host [13-17]". Tras descartar un conflicto de puertos con la integración ZHA, se confirma que el coordinador que está usando es un Sonoff ZBDongle-E.

Рекомендація (не підтверджено)

El error se debe a que el firmware del dongle Zigbee (Sonoff ZBDongle-E) es incompatible con la versión actual del driver Ember que utiliza Zigbee2MQTT. La solución es actualizar el firmware del dongle. Se proporcionan dos herramientas web para realizar el flasheo desde un PC (compatible con Ubuntu/Chrome): 1. La herramienta no oficial de `darkxst`: https://darkxst.github.io/silabs-firmware-builder/ 2. La herramienta oficial de Sonoff: https://dongle.sonoff.tech/sonoff-dongle-flasher/ Actualizando la pila Zigbee a una versión 7.X o superior debería resolverse el problema.
