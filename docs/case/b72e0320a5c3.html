Configuración de triple NAT con OPNsense y Wireguard

Проблема

Un usuario describe una configuración de red con triple NAT (router de compañía, router neutro, OPNsense) y Wireguard funcionando, preguntando si esta "barbaridad" es conceptualmente correcta o problemática.

Рішення

La configuración de múltiples routers en serie, incluyendo doble o triple NAT, funciona perfectamente y es una práctica común, especialmente para mejorar la cobertura WiFi con sistemas Mesh. Si bien puede parecer compleja, es funcional y permite levantar servicios como Wireguard sin problemas, siempre que se configuren correctamente las redirecciones de puertos en cada router. Un usuario confirmó haber configurado exitosamente una triple NAT con OPNsense y Wireguard, redirigiendo los puertos necesarios. Aunque algunos usuarios critican la doble NAT por considerarla "antihigiénica" o generadora de problemas para clientes con una sola FTTH, otros defienden su funcionalidad si se conocen los pasos para redirigir puertos. Como alternativas a la doble NAT, se mencionan el uso de PPPoE para conectar routers en cascada o la configuración de CGNAT para protección.
