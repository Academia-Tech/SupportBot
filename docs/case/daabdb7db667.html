Неактивна опція "Post-filter" при аналізі логів вібрацій в Mission Planner

Case daabdb7db667 · ua_ardupilot

#logging#post-filter#FFT#IMU_RAW#Fast IMU#LOG_BITMASK#вібрації#ArduPilot 4.5+

Проблема

При аналізі логів для налаштування фільтрів вібрацій, користувач виявив, що опція "Post-filter" у вікні "FFT" є неактивною (сірою), що не дозволяє побачити результат роботи фільтрів.

Рішення

Проблема полягає в неправильному режимі логування IMU. Для активації "Post-filter" аналізу необхідно: 1. У налаштуваннях логування (параметр `LOG_BITMASK`) вимкнути `RAW IMU` та увімкнути `Fast IMU` (або `IMU_RAW`). 2. `Fast IMU` (raw logging) записує сирі дані з гіроскопа з високою частотою, що необхідно для аналізу FFT. Старіший метод `Batch logging` (`INS_LOG_BAT_OPT = 4`) усереднює дані і не підходить для цього типу аналізу. Після зміни налаштувань логування та запису нового логу, опція "Post-filter" стане активною.
