Troubleshooting MQTT rc -4 timeout error when connecting Tasmota to AWS IoT

Case deb397e5e413 · tasmota

#aws iot#mqtt#tls#timeout#rc -4#wemos d1#user_config_override.h#authentication

Проблема

A user is trying to connect a Wemos D1 running a custom Tasmota build to AWS IoT using username/password authentication. The device fails to connect with an `MQTT rc -4` (timeout) error, despite the credentials working with other MQTT clients. The user shared their `user_config_override.h` file for review.

Рекомендація (не підтверджено)

Several suggestions were provided for troubleshooting. One user suggested increasing the `MqttWifiTimeout` value (e.g., to 2000), but this did not solve the problem. Another user hypothesized that AWS might require a special registration process for devices with non-standard or unknown ciphers, similar to how Microsoft handles it, and recommended searching the AWS documentation for such a procedure to authorize the device.
