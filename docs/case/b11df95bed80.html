Проблема

Користувач запитав про методи боротьби зі сторонніми DHCP-серверами, які можуть з'являтися в мережі, наприклад, через неправильне підключення обладнання після відновлення електропостачання.

Рекомендація (не підтверджено)

Для боротьби зі сторонніми DHCP-серверами рекомендовано декілька методів: 1. **DHCP Snooping**: Найнадійніший метод, що дозволяє вказати довірені порти, з яких дозволено надходити DHCP-пропозиціям. 2. **Сповіщення**: Використовувати `Dhcp-server alarm` для виявлення неавторизованих серверів. Можна налаштувати скрипт, який буде надсилати сповіщення (наприклад, в Telegram) при спрацюванні цього тригера. Наведено приклад скрипту для відправки повідомлення через Telegram API. 3. **Захист від петель**: Для запобігання загальним проблемам з мережевою топологією, що можуть бути причиною, варто використовувати STP (Spanning Tree Protocol) або `loop detect` на портах.
