Проблема

Un dispositivo Zigbee ("riego doble") se conecta a un router lejano con mala señal (LQI 0/1) en lugar de a uno cercano, causando fallos. Se busca cómo forzar la conexión a un router específico en Zigbee2MQTT sin mover físicamente el dispositivo.

Рекомендація (не підтверджено)

En Zigbee2MQTT, al activar el modo "permitir unirse" (allow join), se puede seleccionar un dispositivo específico para que se una a un router concreto mediante una flecha en la interfaz. Sin embargo, el dispositivo podría cambiar su conexión a otro router con el tiempo. Se plantea la posibilidad de desactivar temporalmente otros routers para forzar la conexión, pero no se confirma si es un método viable o recomendado.
