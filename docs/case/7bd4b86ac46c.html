Error de comunicación Modbus con anemómetro en ESPHome

Case 7bd4b86ac46c · domotica_es

Проблема

Un usuario reporta un error `Modbus command to device=1 register=0x00 no response received` en ESPHome al intentar leer datos de un anemómetro conectado a un ESP32 vía RS485. El sistema había funcionado previamente pero ha dejado de hacerlo.

Рекомендація (не підтверджено)

Se proporciona una guía de troubleshooting detallada. El proceso de diagnóstico avanza por varias etapas: 1. **Análisis de logs inicial:** El error "no response received" con la función Modbus `0x04` (Read Input Registers) sugiere que el anemómetro podría usar una función diferente. 2. **Cambio de función Modbus:** Se prueba con la función `0x03` (Read Holding Registers), que según la hoja de datos del dispositivo es la correcta. Se proporciona una configuración completa de ESPHome para implementarlo. 3. **Nuevos errores:** A pesar de usar la configuración correcta, persisten los errores, esta vez del tipo "Modbus CRC Check failed!". 4. **Diagnóstico final:** La causa más probable de los fallos de CRC es el hardware. Se concluye que el módulo conversor RS485 a TTL utilizado probablemente carece de los pines de control de dirección (RE/DE), lo que provoca fallos de temporización en la comunicación. La recomendación final es sustituir el conversor por uno que sí incluya estos pines para una gestión correcta de la transmisión y recepción.
