Проблема

Un usuario experimentado con el ecosistema Synology busca un nuevo NAS de 8 bahías. Está descontento con la nueva política de Synology que obliga a usar sus propios discos duros, que considera de precio excesivo. Busca alternativas como Asustor, Ugreen u otras marcas. El usuario se define como "básico", no utiliza Docker ni la línea de comandos y valora la facilidad de uso y la robustez del software al que está acostumbrado.

Рекомендація (не підтверджено)

Varios usuarios coinciden en que, para un perfil no técnico que valora un ecosistema de software pulido y fácil de usar, Synology sigue siendo superior. Sin embargo, el usuario principal expresa frustración con las nuevas políticas de Synology respecto a la restricción de discos duros de terceros y sus altos precios, lo que considera un "dislate" y una razón para buscar alternativas. Las alternativas tienen puntos fuertes pero también carencias: - **Synology:** Se recomienda seguir con la marca, pero buscando un modelo de segunda mano o anterior que no tenga las limitaciones de discos. Se destaca su software completo y fácil de usar. - **Qnap:** Se menciona que sus aplicaciones no son tan completas como las de Synology y hay preocupaciones sobre fallas de seguridad y ransomware. Sin embargo, un usuario recomienda el modelo TS-464 y destaca su amplio abanico de aplicaciones y funcionalidades. - **Asustor:** Se considera un buen balance, con apps nativas como Nextcloud y AiFoto, aunque algunas requieren más configuración. Se elogia por no tener el historial de ransomware de QNAP ni las limitaciones de hardware de Synology, e incluso permite cambiar el sistema operativo. Se recomienda probar su demo en línea. - **Ugreen:** Se valora su hardware potente, pero se señala que le faltan aplicaciones nativas y que el software aún necesita madurar, aunque es bueno para Docker. - **General:** Se sugiere que si el usuario no quiere aprender Docker o terminal, Synology sigue siendo la opción más sencilla. Sin embargo, otro usuario con experiencia en Synology y Docker sugiere que aprender Docker facilita la migración entre marcas y reduce la dependencia del software nativo del NAS. Se recomienda esperar a ver si el software de las nuevas marcas mejora.
