Tasmota Rule Disables Itself After Power Loss

Case f30f59570c55 · tasmota

#tasmota#rules#power loss#persistence#system#boot#savedata#ds18b20

Проблема

A user reports that their Tasmota rule, which controls a boiler valve based on DS18B20 temperature sensor readings, works correctly after a soft reboot but its state becomes 'off' after the device loses power and is plugged back in. The user is looking for a way to ensure the rule remains active after a power cycle.

Рекомендація (не підтверджено)

Several users provided advice, noting that rules should persist through power cycles if saved correctly. The primary recommendations were: 1. Use a `system#boot` trigger to explicitly enable the rule(s) on startup. For example: `Rule1 on system#boot do Rule2 1 endon`. 2. Ensure the rule is saved correctly and that `Rule<x> ON` is executed in the console to enable it. 3. Check for syntax errors in the rule itself. **Update:** The user clarified their setup involves controlling a boiler valve with DS18B20 sensors. They confirmed the rule works after a soft reboot but its state changes to `off` after a complete power loss. A new diagnostic step was suggested: check the output of the `savedata` command in the console to verify if the rule's enabled state is being persisted correctly to the device's flash memory.

```rule1
ON DS18B20-3#temperature DO event t1=%value% ENDON
ON DS18B20-1#temperature DO event t2=%value% ENDON
ON event#t2>%mem3% DO var1 1 ENDON
ON event#t2<=%mem3% DO var1 0 ENDON
ON event#t1 DO Backlog var2 %value%; add2 1 ENDON
ON event#t1 DO Backlog var3 %value%; add3 2 ENDON
ON event#t2>%var3% DO Power2 %var1% ENDON
ON event#t2<%var2% DO Power2 0 ENDON```
