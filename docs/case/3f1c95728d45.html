Security risks of exposing Node-RED to the public internet

Case 3f1c95728d45 · tasmota

#Node-RED#security#port forwarding#VPN#authentication#home automation#best practices

Проблема

A user shared a link to their Node-RED instance, which was publicly accessible over the internet without any authentication. Other users immediately pointed out the severe security risks this posed.

Рішення

Multiple users strongly advised against exposing an unsecured Node-RED instance to the internet, calling it a "security nightmare" that allows anyone to manipulate the server and connected devices. The recommended solutions were: 1. Secure the Node-RED instance with a password, as detailed in the official documentation: `https://nodered.org/docs/user-guide/runtime/securing-node-red`. 2. If remote access is not needed, disable port forwarding in the router to keep the instance on the local network. 3. For secure remote access, consider using a VPN. The user acknowledged the advice and stated they would implement the security measures.
