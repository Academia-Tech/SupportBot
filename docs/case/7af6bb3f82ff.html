Проблеми з прошивкою ArduPilot на Speedybee F405 v3 та підключенням до Mission Planner

Case 7af6bb3f82ff · ua_ardupilot

#ArduPilot#Speedybee F405 v3#прошивка#Mission Planner#ESC#DFU#STM32CubeProgrammer#Inav#COM-порт#бутлоадер

Проблема

Користувач, який переходить з Betaflight, прошив Speedybee F405 v3 файлом arducopter_with_bl.hex з офіційного сайту ArduPilot. Mission Planner читає дані, але при підключенні батареї звук видають лише ESC, налаштування не зберігаються, і з'являється помилка "зняти пропи". Спроби прошивки через Inav Configurator та STM32CubeProgrammer, а також повне стирання флешу, не вирішили проблему. Після прошивки ArduPilot Mission Planner не підключає польотник.

Рекомендація (не підтверджено)

Було запропоновано кілька рішень: 1. Перепрошити через Inav Configurator. 2. Використати STM32CubeProgrammer для прошивки та повного стирання флешу. 3. Спробувати зібрати прошивку 4.7.0 через custom.ardupilot.org/add_build. 4. Перевірити, чи не змінився COM-порт після прошивки. 5. Зазначено, що _bl.hex містить бутлоадер, що дозволяє подальші оновлення через Mission Planner. 6. Пояснено, що польотний контролер сам не видає звуків, а ESC не пілікають вдруге, якщо FC не знає про їхнє розташування (потрібно заливати пресет через Mission Planner без батареї). Незважаючи на спроби, проблема з коректною роботою ArduPilot та підключенням до Mission Planner залишається невирішеною.
