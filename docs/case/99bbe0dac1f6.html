DS18B20 Temperature Sensor Randomly Fails on Sonoff 4ch Pro After Power Cycle

Case 99bbe0dac1f6 · tasmota

#DS18B20#Sonoff 4ch Pro#GPIO#temperature sensor#power cycle#SetOption74#pull-up resistor#troubleshooting

Проблема

A user with a Sonoff 4ch pro and an external DS18B20 temperature sensor connected to GPIO2 experienced random sensor failures after power cycling. The issue occurred across multiple identical devices, requiring another power cycle to restore functionality. The setup used the internal pull-up resistor (`SetOption74 1`) and the `tasmota-sensors.bin` firmware.

Рішення

The issue was resolved by moving the DS18B20 sensor from GPIO2 to GPIO1. The user confirmed this fixed the problem, with the sensor working reliably for over two days (confirmed with '👍👍'). Other suggestions during troubleshooting included: - Using a physical 4.7kΩ pull-up resistor instead of the internal one. - Avoiding GPIO0 and GPIO2 for sensor signals as they can be problematic during boot. GPIO1 (TX) or GPIO3 (RX) were suggested as alternatives. - A link to the official documentation was also shared: https://tasmota.github.io/docs/DS18x20/#multiple-sensors
