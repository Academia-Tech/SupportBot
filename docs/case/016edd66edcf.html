Acceso a NAS como 'no seguro' con VPN y DDNS

Case 016edd66edcf · naseros

#NAS#VPN#WireGuard#HTTPS#certificado#seguridad#DDNS#Asustor

Проблема

El usuario accede a su NAS (Asustor) a través de un DDNS del router Asus y una VPN (WireGuard), pero el navegador muestra la conexión como 'no segura' a pesar de tener HTTPS forzado y un certificado válido. Esto ocurre incluso cuando la VPN está conectada o en acceso local, pero no cuando se accede sin VPN.

Рекомендація (не підтверджено)

Para solucionar la advertencia de 'sitio no seguro' al acceder a un NAS, incluso a través de VPN, se proponen dos soluciones. La principal es que el problema se debe a la falta de un certificado de confianza. Se recomienda generar certificados válidos y gratuitos usando Let's Encrypt para que el navegador reconozca el sitio como seguro. Alternativamente, se pueden crear certificados autofirmados, que ofrecen la misma seguridad de cifrado pero seguirán mostrando una advertencia en el navegador que el usuario debe aceptar manualmente. Se aclara que al usar una VPN, el acceso es local y el navegador puede mostrar la advertencia si el certificado no es de confianza o es autofirmado. Se proporcionó también un enlace a un vídeo de YouTube (https://www.youtube.com/watch?v=GO9Ji7Nslj0) con información adicional.
