Crear un disparador (trigger) en Home Assistant para el timbre de Ring Intercom

Case 26d6ff619ef1 · domotica_es

#Home Assistant#Ring Intercom#automatización#trigger#YAML#notificaciones#timbre

Проблема

Un usuario con la integración oficial de Ring en Home Assistant no sabe cómo crear una automatización que se dispare cuando alguien llama al timbre del Ring Intercom, ya que el sensor de última actividad no es fiable.

Рішення

La solución es utilizar un disparador de estado (`state trigger`) en la automatización que vigile la entidad del evento del timbre. Se proporciona el siguiente código YAML como ejemplo para el disparador: ```yaml - trigger: state entity_id: - event.verja_timbre from: null ``` (Nota: `event.verja_timbre` debe reemplazarse por la entidad correcta). Además, se aportan casos de uso para esta automatización: se puede capturar una imagen de una cámara y enviarla por Telegram, o crear un botón en la pantalla de bloqueo del móvil para abrir la puerta. Esto último se menciona como una ventaja sobre la app de Ring, que según un usuario no tenía widget (aunque otro usuario indica que ahora sí podría tenerlo).
