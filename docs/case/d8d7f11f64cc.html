Relay on D1 Mini v3 Not Responding to Toggle Command in Tasmota

Case d8d7f11f64cc · tasmota

#D1 Mini#relay#toggle#GPIO#template#module#configuration#backlog

Проблема

A user with a Wemos D1 mini v3 running Tasmota v12.2.0 was unable to control a relay connected to a GPIO. Clicking the "Toggle" button in the web UI had no effect on the pin's state. The user's configuration was requested to diagnose the problem.

Рішення

The initial diagnosis suggested an incorrect device template, and a `backlog` command with a standard `Relay` component was provided. When this failed, further troubleshooting revealed the true cause. Using a multimeter, the user confirmed the GPIO pin was switching voltage correctly, isolating the problem to the relay module. The module was identified as a 'low-level trigger' type, which activates on a LOW signal (0V). The final, correct solution was to modify the Tasmota template to use the `Relay_i` (inverted) component instead of `Relay`. This change accommodates the relay's inverted logic, and the user confirmed it resolved the issue.
