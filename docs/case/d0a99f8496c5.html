Troubleshooting Connection of Multiple DS18B20 Sensors to an ESP8266

Case d0a99f8496c5 · tasmota

#DS18B20#ESP8266#D1 mini#GPIO#multiple sensors#ESP32#power supply#XIAO ESP32C3

Проблема

A user was experiencing issues connecting more than four waterproof DS18B20 temperature sensors to a D1 mini (ESP8266). The sensors were connected in a star topology on GPIO0 with a 4.7kΩ pull-up resistor and a stabilized 5V power supply. The user was unsure if the problem was electrical (power drop) or a software race condition.

Рішення

The primary solution for an unstable DS18B20 sensor was to avoid using GPIO0, which has special functions during boot. A user confirmed their issue was resolved by moving the sensor to a different GPIO. Further discussion added important context for DS18B20 stability, especially over long cables: * **Voltage and Cable Length:** The official documentation suggests a limit of <1m cable length when powering the sensor with 3.3V. For longer runs, 5V power is recommended, which can support 20m+ cables. * **Logic Level Conversion:** When using 5V to power the sensor with a 3.3V microcontroller (like ESP8266), a Logic Level Converter (LLC) is advised to protect the MCU's GPIO pins, although some users note the IOs may be 5V tolerant. * **Real-world Examples:** Users shared successful implementations with cables of 3m, 5m, and even over 10m on 3.3V, suggesting it can work if electrical noise is low. * **Advanced Solution for Long Runs:** A user shared a robust configuration for a 20-meter line with 6 sensors: power the sensors (VCC) with 5V, but use an 820 ohm pull-up resistor from the data line to the 3.3V supply of the microcontroller. This provides a stronger pull-up than the typical 4.7kΩ and was successful for a long, multi-sensor bus.

GPIO15 is always pulled low, so you can’t use the internal pull-up resistor. You have to keep this in mind when using GPIO15 as an input to read a switch or connect it to a device with an open-collector (or open-drain) output, like I²C.
GPIO0 is pulled high during normal operation, so you can’t use it as a Hi-Z input.
GPIO2 can’t be low at boot, so you can’t connect a switch to it.
