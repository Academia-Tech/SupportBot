Способи віддаленого доступу до роутера MikroTik без публічної IP-адреси

Case a857d6cda968 · mikrotik_ua

#remote access#NAT#VPN#Back to Home#CHR#SSTP#Wireguard#ZeroTier

Проблема

Користувач запитує, як можна підключитися до роутера MikroTik для керування, якщо у нього немає виділеної публічної IP-адреси (знаходиться за NAT провайдера).

Рекомендація (не підтверджено)

Запропоновано декілька рішень для доступу до роутера за NAT: 1. **IP/Cloud (DDNS):** Якщо IP-адреса публічна, але динамічна. 2. **Back to Home (BTH):** Вбудована функція в RouterOS 7 (тільки для архітектур ARM/ARM64), яка працює навіть за NAT провайдера. 3. **VPN до хмарного сервера (CHR):** Налаштування VPN-тунелю (SSTP, Wireguard) від роутера до власного сервера CHR з публічною IP. Це найнадійніший варіант для керування. 4. **ZeroTier:** Сервіс для створення віртуальних мереж, пакет доступний для RouterOS на ARM/ARM64. Для пристроїв на архітектурі MIPS, де BTH та ZeroTier недоступні, найкращим варіантом є CHR + Wireguard.
