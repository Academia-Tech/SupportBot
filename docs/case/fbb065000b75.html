Unstable Input Signal on Wemos D1 from a Commercial Relay's Auxiliary Contact

Case fbb065000b75 · tasmota

#tasmota#wemos d1#gpio#input#noise#debounce#capacitor#ghost switching#relay feedback

Проблема

A user is trying to get feedback from a commercial relay's auxiliary contact, which is connected to an input on a Wemos D1 running Tasmota. The input signal is very unstable, rapidly toggling on and off. The user provided a wiring diagram and asked if the connections are correct.

Рішення

The issue of an unstable input signal from a relay's auxiliary contact, likely caused by 50/60Hz electrical noise, was resolved after the user received several suggestions and confirmed their helpfulness. The recommended solutions include: 1. **Hardware Filtering:** Add a 10µF capacitor between the input pin and ground. A 10kΩ pull-up resistor from the input pin to the 3.3V line was also suggested. 2. **Software Debouncing:** Increase the debounce timer in Tasmota, referencing the documentation on 'ghost switching' for guidance. 3. **Software Pull-up:** While `pinMode(D1,INPUT_PULLUP);` was mentioned, it was noted this might not be supported in Tasmota.
