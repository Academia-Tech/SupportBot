Configurar un modo "No Molestar" para notificaciones de Home Assistant por usuario

Case 6dcf9f65b29a · domotica_es

#home assistant#notificaciones#no molestar#android#oneplus#automatización#condición de tiempo

Проблема

Un usuario pregunta si es posible configurar un modo "No Molestar" por usuario en Home Assistant para que las notificaciones no suenen en un horario determinado (ej. de 21:30 a 8:30). El usuario intuye que deberá configurarlo a nivel de la aplicación en Android, pero busca confirmación y alternativas.

Рішення

Se confirma que Home Assistant no tiene una función nativa de "No Molestar" por usuario. Se proponen dos soluciones que el usuario agradece: 1. **A nivel de Home Assistant:** Se puede simular añadiendo una condición de tiempo a las automatizaciones que envían notificaciones, para que solo se envíen dentro de un horario permitido (p. ej. entre las 8:30 y las 21:30). 2. **A nivel del Sistema Operativo (Android):** Es la solución más directa. Para teléfonos OnePlus con OxygenOS (Android 13/14), el usuario `user_edae2508a6` indica que la ruta específica es `Ajustes → Notificaciones y barra de estado → No molestar → Programar`. Esto permite silenciar las notificaciones de la app de Home Assistant en el horario deseado.

En Home Assistant de momento no hay una función nativa para establecer un "modo no molestar" por usuario (por ejemplo, para silenciar notificaciones entre ciertas horas).

Lo más habitual es hacerlo a nivel del sistema operativo, como bien dices, configurando el modo "No Molestar" en Android o iOS.

Dicho esto, sí puedes simularlo desde HA:

Puedes crear una condición por horario o usar una input_boolean "modo no molestar" y, si está activado, evitar que se envíe la notificación.

También puedes filtrar por usuario si envías notificaciones usando servicios como notify.mobile_app_nombre_usuario.

Pero eso solo impide que Home Assistant las mande, no que el teléfono las silencie si ya las recibe. Así que, efectivamente, lo más directo sigue siendo hacerlo desde Android.
