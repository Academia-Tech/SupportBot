Diagnóstico de alto consumo de CPU en un Mini PC N100 con Home Assistant

Case 5ef5f0f978c4 · domotica_es

#cpu#rendimiento#n100#mini pc#diagnóstico#top#htop#systemmonitor

Проблема

Un usuario con un Mini PC N100 nota un uso de CPU continuo y elevado, a pesar de tener solo Home Assistant, Adguard y unos 70 dispositivos. Pregunta si es un comportamiento normal. Otro usuario confirma que no lo es, ya que su sistema similar en un N100 apenas consume un 1% en reposo.

Рішення

La solución fue un proceso de diagnóstico para identificar el proceso o complemento culpable. Se recomendó usar herramientas de monitorización del sistema. El usuario utilizó el comando `top` en la terminal, lo que le dio la pista para encontrar un complemento que estaba consumiendo casi todo el núcleo de la CPU. Tras desinstalarlo, el consumo se normalizó. Otro usuario añade la hipótesis de que algunos hipervisores pueden reservar un porcentaje de CPU para uso futuro, aunque no la estén usando activamente, lo que podría reflejarse en las estadísticas generales.
