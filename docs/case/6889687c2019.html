Reading PWM Signal Duty Cycle as an Input in Tasmota

Case 6889687c2019 · tasmota

#PWM#duty cycle#input#ESP32#Tasmota#feature request#sensor

Проблема

A user inquired if it's possible to use an ESP32S3 running Tasmota to measure the duty cycle of an incoming PWM signal.

Рішення

It was confirmed that Tasmota does not have a built-in feature to read the duty cycle of an incoming PWM signal. However, a software solution using a custom Berry script to count pulses was discussed and its effectiveness was confirmed by a user who stated, "This absolutely works, it's working in all my radio controlled boats, even with 8 channels versions." The potential of this solution was also acknowledged by another user who planned to discuss its implementation into Tasmota.

const int pwmPin = 2; // GPIO-Pin, an dem das PWM-Signal anliegt
volatile unsigned long startTime;
volatile unsigned long endTime;
volatile bool pulseStarted = false;

void ICACHE_RAM_ATTR onInputChange()
{
if (digitalRead(pwmPin) == HIGH)
{
startTime = micros();
pulseStarted = true;
}
else
{
if (pulseStarted)
{
endTime = micros();
pulseStarted = false;
}
}
}

void setup()
{
pinMode(pwmPin, INPUT);

attachInterrupt(digitalPinToInterrupt(pwmPin), onInputChange, CHANGE);

Serial.begin(9600);
}

void loop()
{
if (!pulseStarted)
{
float dutyCycle = (endTime - startTime) * 100.0 / (endTime - startTime + 200); // 200 ist die Periode des PWM-Signals in Mikrosekunden
Serial.print("Duty Cycle: ");
Serial.print(dutyCycle);
Serial.println("%");
}

delay(1000);
}
