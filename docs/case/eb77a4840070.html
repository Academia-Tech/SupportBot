Dispositivos Zigbee2MQTT no disponibles periódicamente

Case eb77a4840070 · domotica_es

#Zigbee2MQTT#Z2M#Zigbee#Home Assistant#Conectividad#Diagnóstico#Problemas#Addon

Проблема

Un usuario experimenta que sus dispositivos Zigbee2MQTT aparecen como 'unavailable' cada 24-36 horas, y el addon de Z2M se detiene. Ha implementado una solución temporal reiniciando el servicio, pero sospecha que un dispositivo Zigbee defectuoso podría ser la causa.

Рішення

Para solucionar el problema de los dispositivos Zigbee2MQTT que se vuelven 'unavailable', se siguieron varias recomendaciones que resultaron útiles. El diagnóstico incluye: 1. Revisar los logs de Z2M para identificar errores. Se sugiere usar una IA como GPT para interpretar los logs, lo cual ha sido considerado una muy buena idea por los usuarios. 2. Asegurar la conexión física del coordinador: si es USB, usar un alargador; si es Ethernet (tipo SLZB), revisar que no se desconecte y añadir un sensor de ping para monitorizarlo. 3. Considerar el uso de un 'watchdog' (aunque no siempre funciona bien en Z2M) o automatizaciones para reiniciar el addon con un trigger de sensor 'unavailable'. Es crucial identificar la causa raíz del problema.
