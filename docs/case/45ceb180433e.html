Configuración de red con doble NAT, Pi-hole y WireGuard en router de ISP

Case 45ceb180433e · naseros

#doble NAT#Pi-hole#WireGuard#router ISP#DHCP#redes#configuración de red#AP

Проблема

Un usuario describe una configuración de red compleja con un router principal del ISP que no permite cambiar el DNS, y un router secundario. Al poner el router secundario en modo AP, Pi-hole deja de funcionar porque no puede configurar el DNS en el router principal. Además, tiene problemas para que WireGuard funcione en esta configuración de doble NAT.

Рекомендація (не підтверджено)

Se proponen varias soluciones arquitectónicas para resolver el conflicto de doble NAT y la limitación del router del ISP: 1. Desactivar el servidor DHCP en el router principal y utilizar el servidor DHCP integrado en Pi-hole (corriendo en modo 'host' en Docker). 2. Poner el router secundario en modo AP (el usuario ya lo tiene así) y buscar una forma de forzar el DNS en la red. 3. Investigar si es posible desactivar el NAT en el router principal (modo bridge/monopuesto) y delegar las funciones de NAT y DHCP completamente al router secundario.
