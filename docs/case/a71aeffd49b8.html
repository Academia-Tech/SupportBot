Creating a Watchdog or Failsafe Timer Reset by WebSend Commands

Case a71aeffd49b8 · tasmota

#watchdog#failsafe#rules#websend#timer#ruletimer#pulsetime#automation

Проблема

A user wanted to create a watchdog timer that would be reset every time a `WebSend` command is received. If no command is received within a certain period, a failsafe action (turning off a relay) should be triggered. The user was looking for an event that fires on any `WebSend` command.

Рішення

A solution was devised using Tasmota Rules. Instead of a generic event, the `WebSend` command should set a specific variable (e.g., `Websend [ipaddress] event var1=1`). A rule can then be created to react to this variable change and reset a timer: `on var1#state do ruletimer1 600 endon` Another rule triggers the failsafe when the timer expires: `on rules#timer=1 do power1 0 endon` An alternative, simpler solution was also suggested: using the `PulseTime` command. This command can set a temporary auto-off timer for the relay, which is re-triggered if the command is sent again, effectively acting as a watchdog.
