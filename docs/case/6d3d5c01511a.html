Troubleshooting Random Switch Triggering on Shelly 1 Gate Controller

Case 6d3d5c01511a · tasmota

#shelly 1#tasmota#gpio#ghost switching#electrical noise#troubleshooting#gate controller

Проблема

A user described a random issue with a Tasmota-flashed Shelly 1 used as a gate controller. When the gate's open/close sensor (Switch1) is activated, the doorbell sensor (Switch2 on GPIO3) is also triggered intermittently. The problem does not occur if the wires for Switch2 are disconnected.

Рішення

The issue was resolved by the user. They abandoned the custom wiring on GPIO3 and instead used the official Shelly Add-on for the Shelly 1. This hardware add-on provided a stable way to connect the external sensors, eliminating the random triggering problem. The user also provided a link to the new model of the add-on: `https://www.shelly.com/en/products/shop/shelly-plus-add-on`. Initial diagnostic suggestions included checking for hardware problems or device resets, and noting that using external GPIOs like GPIO3 for anything other than programming is an unsupported configuration.
