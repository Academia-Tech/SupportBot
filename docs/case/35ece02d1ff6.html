Проблема

Users discuss the difficulty of getting consumer-grade 2D printers, especially from HP and Samsung, to work reliably with Linux. They compare experiences with Windows and note that even when using the standard CUPS printing system, vendor support and driver quality can be lacking, leading to frustrating issues.

Рекомендація (не підтверджено)

For reliable Linux compatibility, used Kyocera office printers are recommended as they are reported to work well with CUPS. In general, office-grade or business-grade printers (e.g., HP LaserJet business line) tend to have better support and reliability on Linux than consumer-grade models. Further details on the driver situation: Both CUPS and Windows are moving away from legacy drivers towards 'driverless' protocols like AirPrint and IPP Everywhere. CUPS is removing legacy driver support from the main application, requiring old drivers to be wrapped in a PAPPL application (https://www.msweet.org/pappl/) to continue working. Most open-source drivers have already been ported. This shift aims to improve security and stability by moving drivers out of the kernel space. While this may cause issues for older printers, modern 'driverless' printers should work seamlessly across platforms.

In practice nothing. They're removing support for legacy drivers in CUPS itself, keeping only support for "driverless" printers (basically AirPrint). The old drivers can still work with CUPS but need to be turned into their own application using PAPPL which CUPS will talk to using its supported protocols https://www.msweet.org/pappl/
