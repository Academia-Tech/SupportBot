Troubleshooting RC522 RFID Reader on ESP32-CAM with Tasmota

Case 9e7e2f803f55 · tasmota

#Tasmota#ESP32-CAM#RC522#RFID#troubleshooting#GPIO#boot issue

Проблема

A user was experiencing crashes and detection failures when trying to use an RC522 RFID reader with an ESP32-CAM module flashed with Tasmota. The system would either crash with an 'Illegal instruction' error or fail to detect the reader.

Рекомендація (не підтверджено)

The user initially suspected wiring issues (GPIO12 for RST) and corroded connectors were causing the RC522 RFID reader to fail on an ESP32-CAM. The root cause was later identified as a software conflict: the MFRC522 Arduino library defaults to the VSPI bus, which conflicts with the camera on the ESP32-CAM board. The initial workaround was to modify the library to use the HSPI bus instead. An official fix has since been introduced by arendst into the Tasmota development branch, which provides a clean way for the MFRC522 to use the proper SPI pins without manual library modification.

I found the reason, why it does not work:
"Important: If your micro controller supports multiple SPI interfaces, the library only uses the **default (first) SPI** of the Arduino framework."

By default one of the two available SPI busses is used: VSPI - but this bus pins are used for communicating to the camera :/
The other SPI bus makes use of GPIO 12 to 15 (HSPI) - but is not used by the MFRC522 Arduino library :/

Oh, a quickfix in the MFRC522 driver made it work:
I replaced
SPI.begin();
by
SPI.begin(SCK_PIN, MISO_PIN, MOSI_PIN);
(and some compile time checks so if I forget some #define it would tell me - and some fallback to SPI.begin() )
