Using RC522 RFID Reader with Tasmota on ESP32-CAM with SPI Pin Conflicts

Case aab5098ec089 · tasmota

#ESP32-CAM#RC522#RFID#SPI#Tasmota#driver#custom firmware#pin conflict

Проблема

A user was trying to connect an RC522 RFID reader to an ESP32-CAM module running Tasmota. The device would either crash or fail to detect the reader. The issue was traced to a conflict with the SPI bus, as the default SPI pins (VSPI) on the ESP32-CAM are used by the camera module.

Рішення

The user initially solved the problem by modifying the Tasmota driver for the MFRC522 (`xsns_80_mfrc522.ino`), replacing the generic `SPI.begin()` call with a version that specifies the pins: `SPI.begin(Pin(GPIO_SPI_CLK), Pin(GPIO_SPI_MISO), Pin(GPIO_SPI_MOSI));`. This forced the library to use the SPI pins configured in the Tasmota WebUI. **Update:** A user later reported that a cleaner, official solution has been identified by arendst and introduced into the development branch of Tasmota. This change ensures the MFRC522 uses the proper SPI pins on the ESP32 without requiring manual code modification.

I found the reason, why it does not work:
"Important: If your micro controller supports multiple SPI interfaces, the library only uses the **default (first) SPI** of the Arduino framework."

By default one of the two available SPI busses is used: VSPI - but this bus pins are used for communicating to the camera :/
The other SPI bus makes use of GPIO 12 to 15 (HSPI) - but is not used by the MFRC522 Arduino library :/

(and some compile time checks so if I forget some #define it would tell me - and some fallback to SPI.begin() )
