Guide for flashing a WVC-1600 grid tie solar inverter with Tasmota

Case 7e61e319f48f · tasmota

Проблема

A user shared a detailed, self-documented guide on how to replace the original communication module of a WVC-1600 grid tie solar inverter with an ESP-07 running Tasmota. The goal is to gain local control and read data via MQTT without relying on the manufacturer's cloud.

Рекомендація (не підтверджено)

The user provided a comprehensive guide for modifying a WVC-1600 inverter. This involves replacing the original WiFi module with an ESP-07 (with an added capacitor for power stability). To initialize the inverter without the cloud, a specific sequence of serial commands must be sent at 115200 baud (e.g., `+ILOPCONNECT=WIFI_CONNECT`, `+ILOPCONNECT=SERVER_CONNECT`, `+ILOPDATA=ICA,PowerSwitch,1`). Once initialized, the inverter sends telemetry data via serial, which Tasmota can forward to MQTT. The user also provided a Tasmota rule for automating the initialization on boot: `Rule1 ON System#Boot DO Backlog Baudrate 115200; SerialSend1+ILOPCONNECT=WIFI_CONNECT; SerialSend1+ILOPCONNECT=SERVER_CONNECT; SerialSend1+ILOPDATA=ICA,PowerSwitch,1 ENDON`. They also shared pinout details and warned that an ESP32 might draw too much power.

It runs the default Tasmota firmware, has RX and TX pins configured (I did not yet connect TX since there is no need for me to send the three known commands to it (on/off, power limit 1..100%, and enter/exit some weird "Energy_Cleared" state)
If someone wants to do this: to initialize the inverter controller, so it doesn't just send "AT+AWSSRESET" you either need to
* connect the inverter to WIFI & Internet first (requires app download + app account creation), then you may need to turn it on via the app interface (by default the inverter may be disabled!) then replace the original module
* or remove the WIFi communication module (the green board) and send the following strings via serial (115200 baud, e.g. from a computer via a USB-serial adapter or via Tasmota Web console command "SerialSend1 [string here]") to the Inverter, in the given order. Each string terminated by a newline character [Notes are in brackets]:
1) +ok [because the inverter controller will send requests/commands to the WIFI module which probably should be answered. All commands only need to be answered by "+ok". Yes, all WiFi module communication is prefixed by a '+']
2) [empty string, so newline character only. maybe this is not needed and just fucked up firmware...]
3) +ILOPCONNECT=BLE_START
4) [empy string again - try skipping it for science]
5) +ILOPCONNECT=WIFI_CONNECT
6) [empy string again - try skipping it for science]
7) +ILOPCONNECT=SERVER_CONNECT
8) [empy string again - try skipping it for science]
9) +ILOPDATA=ICA,PowerSwitch,1
* you may as well try to only send 5), 7) and 9). For science [insert GlaDOS meme here]

Then the inverter will send two strings alternatingly:
AT+SENDICA=property,PV_Volt,19.0,PV_Current,0.00,PV_Power,0.0,AC_Volt,0.0,AC_Current,0.00,Out_Power,0.0,Temperature,34.0,Power_adjustment,100,Energy,0.00
AT+SENDICA=property,PowerSwitch,0,Plant,0.00,Emission,0.00,Time,30,P_adj,0,TEMP_SET,66

In this case you see "PowerSwitch,0" which means it is powered off/disabled. To power on the inverter, you need to send the string mentioned in 9) - to turn it off, send the same, but replace '0' instead of '1'

Additional note: since the ESP8566 module already requires an additional capacitor to stabilize the power supply (the backside of the green PCB contains a voltage regulator 22V->3.3V), I guess an ESP32 would draw too much power and either not work at all or fry the power supply on the green PCB.

EDIT:
This is probably the rule that covers everything that is needed on boot:
Rule1 ON System#Boot DO Backlog Baudrate 115200; SerialSend1+ILOPCONNECT=WIFI_CONNECT; SerialSend1+ILOPCONNECT=SERVER_CONNECT; SerialSend1+ILOPDATA=ICA,PowerSwitch,1 ENDON
An additional rule may be useful for answering all received (non-empty) strings with "+ok", and value extraction maybe can be done with a further rule, too. Since the trigger SerialReceived#Data did not work for me, I used
ON Time#Minute DO Backlog SerialSend1+ok; SerialSend1+ok ENDON
To at least acknowledge two messages every minute

I encountered my WVC-1600 to sometimes only give no output, so it may be needed to turn it off and back on again via these two console commands:
SerialSend1+ILOPDATA=ICA,PowerSwitch,0
SerialSend1+ILOPDATA=ICA,PowerSwitch,1
from time to time, e.g. by a rule.
