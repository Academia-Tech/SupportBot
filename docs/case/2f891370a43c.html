Two Sonoff S31 plugs fail to boot after flashing custom Tasmota firmware

Case 2f891370a43c · tasmota

#Sonoff S31#flashing#bricked#boot failure#custom firmware#troubleshooting#erase flash#hardware variation

Проблема

A user has 2 out of 20 Sonoff S31 plugs that fail to boot after being flashed with a self-built Tasmota 12.5.0.4 `sensors` firmware (with Prometheus metrics). The plugs flash successfully but then become unresponsive, requiring FTDI recovery. The official OTA `sensors` firmware works correctly, but the custom build fails only on these two specific plugs.

Рекомендація (не підтверджено)

Several troubleshooting steps were suggested. One user proposed that the issue might be due to minor hardware variations and recommended trying a different, but similar, device template like 'BlitzWolf' as a trial-and-error approach. Another user suggested ensuring the flash is completely erased before flashing the new firmware, using a tool like Tasmotizer which has an "erase before flashing" option, to rule out any leftover data causing conflicts. The original user also speculated that shoddy or slightly different components could be the cause.
