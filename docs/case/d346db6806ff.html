El complemento Zigbee2MQTT se detiene y deja los dispositivos no disponibles

Case d346db6806ff · domotica_es

#Zigbee2MQTT#addon#troubleshooting#unavailable#diagnóstico#log

Проблема

Un usuario informa que cada 24-36 horas, el complemento Zigbee2MQTT (z2mqtt) se detiene, lo que provoca que todos sus dispositivos Zigbee aparezcan como "no disponibles" en Home Assistant. El usuario tiene que reiniciar manualmente el complemento para restaurar la funcionalidad. Sospecha que podría ser causado por un dispositivo defectuoso, posiblemente de una marca menos reputada de AliExpress.

Рішення

Se proponen varios métodos de diagnóstico y solución para cuando el complemento Zigbee2MQTT se detiene: 1. **Revisar los logs:** Analizar los logs de Z2M en busca de errores. Como idea novedosa, el usuario user_d19b47257c sugiere pegar los logs en ChatGPT para que ayude a interpretarlos. El usuario que preguntó considera esta idea muy buena y agradece la sugerencia. 2. **Revisar el coordinador:** Si es USB, usar un alargador. Si es Ethernet (tipo SLZB), verificar que no se desconecte (añadir sensor ping para monitorizar). 3. **Automatización de reinicio:** Si se desconecta, usar un watchdog o automatizar el reinicio del addon con un sensor de disponibilidad como trigger.
