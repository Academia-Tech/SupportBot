Tasmota AP Assigns Invalid 169.254.x.y IP Address

Case 399b34870ac8 · tasmota

#flashing#Sonoff Basic#AP mode#DHCP#IP address#169.254.x.y#troubleshooting

Проблема

After flashing a Sonoff Basic with Tasmota, the device enters AP mode as expected. However, when a client (like an iPhone) connects to the Tasmota Wi-Fi network, it receives a self-assigned IP address in the 169.254.x.y range. This indicates that the DHCP server on the Tasmota device is not functioning correctly, preventing access to the web configuration page.

Рекомендація (не підтверджено)

The recommended solution for this issue is to perform a full flash erase of the device and then re-flash the Tasmota firmware. This process clears any potentially corrupted data or settings in the flash memory that could be causing the DHCP server to fail.
