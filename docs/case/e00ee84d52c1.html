Preventing Relay Chatter from Fluctuating Sensor Readings in Tasmota Rules

Case e00ee84d52c1 · tasmota

#tasmota#rules#sensor#hysteresis#relay chatter#timer#ds18b20#solar heater#failsafe

Проблема

A user's solar heater control, based on temperature sensor readings, was causing the relay to turn on and off rapidly ('clicking') due to temperature fluctuations around the setpoint. They asked how to use a timer to reduce the frequency of checks to avoid this.

Рекомендація (не підтверджено)

A user explained that sensor readings often fluctuate. To prevent relay chatter, a simple timer is one option, but a better solution is to implement hysteresis (a deadband) in the rule logic. For example, turning the relay on when temperature rises above `T+1` and off only when it falls below `T-1`. Another user shared a related experience with an analog sensor, where they implemented a failsafe rule to prevent a relay from getting stuck 'on'. Their solution was to add a secondary rule that unconditionally turns the relay off after a set period, acting as a watchdog. For the initial question about a simple timed rule, the syntax `Rule1 ON Time#Minute|10 DO <command> ENDON` was provided, which executes a command every 10 minutes.
