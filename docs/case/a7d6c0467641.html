Блокування доступу до інтернету для IoT пристрою зі збереженням локального доступу

Case a7d6c0467641 · mikrotik_ua

#firewall#iot#isolation#vlan#mikrotik#internet block#arp reply-only#home assistant

Проблема

Користувач хоче заборонити інвертору доступ до Інтернету для запобігання відправки даних, але зберегти можливість керувати ним з локальної мережі через Home Assistant по VPN. Існує побоювання, що пристрій може змінити IP-адресу, щоб обійти блокування.

Рекомендація (не підтверджено)

Рекомендовано декілька підходів для ізоляції пристрою: 1. Прив'язати статичну IP-адресу до MAC-адреси пристрою в налаштуваннях DHCP-сервера. 2. Для запобігання самостійної зміни IP-адреси пристроєм, на інтерфейсі можна увімкнути режим `arp reply-only`. 3. Найбільш надійний метод: створити окрему підмережу (VLAN або bridge) для IoT-пристроїв. 4. Налаштувати правила файрволу: заборонити вихідні з'єднання з цієї підмережі в інтернет (ланцюжок `forward`), але дозволити встановлені та пов'язані з'єднання. Це дозволить ініціювати підключення до пристрою з локальної мережі, але не дозволить пристрою самому встановлювати з'єднання ззовні.
