Using Tasmota with Timers in an Offline Environment Without Wi-Fi

Case 227543e81e99 · tasmota

#offline#no wifi#timer#rtc#real-time clock#ntp#access point#esp8266

Проблема

A user wants to use an ESP8266 with Tasmota in a garage with no network connection. The goal is to use the web interface to control an inverter with a timer, but they need to prevent Tasmota from entering Wi-Fi configuration mode.

Рішення

It was explained that Tasmota's timers are not suitable for this use case because they rely on a network connection to an NTP server to get the correct time after a reboot. Without a network, the device's clock will reset to 00:00, making timers unreliable. The definitive solution is to use an external Real-Time Clock (RTC) module with the ESP8266. For managing the device without a constant Wi-Fi connection, it was suggested to either pre-configure the credentials at home and then use a phone's hotspot with the same SSID/password for occasional access, or to set up a dedicated old router as a local-only access point in the garage. A link to a GitHub discussion on Tasmota's AP mode was also shared: https://github.com/arendst/Tasmota/discussions/13362.
